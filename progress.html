Reflection from workbook two

My experience with 3D using p5 for workbook two could have been a lot better if I was more prepared.

  • Firstly I did not factor in enough time as I didn't realise incorporating 3D would take sigificantly longer time
  • There wasn't a tutorial online or code snippet for the idea I had in mind
  • I didn't have enough time to experiment much before the deadline
  • There wasn't enough documentation on debugging, which was my biggest problem
  • AI was not helpful

Goals for workbook three

I wanted to try make an interactive 3D workbook using Three.js and reduce my reliance on AI. The goal is to create a scene where the viewer can navigate around a scene with iconic tributes to key moments from this subject.

I followed this tutorial by JS Mastery which taught me a few things:

  • Three.js is like a library for WebGL. We can write code for 3D models with much fewer lines of code and complication.
  • Three.js is reccomended to use with React because we can use renderers to help us (React Three Fiber and React Three Drei)

However...I ran into so many problems that I couldn't understand or solve. I couldn't even get my model to show up in the scene. At this point I felt like I was learning 3 new applications at once, and it was not going well. This was when I knew I needed to pivot.

Restarting

Now that I had a rough overview of Three.js, I wanted to use it as an add on to skills I've already learnt. This was when I found Andrew Woan's creative development coding course. Andrew Woan teaches us the fundamental basics of incorporating 3D into vanilla Javascript using Three.js, as well as referencing documentation as we go. This was very similar to Matilda and Mei's teaching methods in class where they taught us to read the p5 documentation together with the live coding session.
